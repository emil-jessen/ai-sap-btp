# my-btp-app ‚Äî Project Context

## What This App Is
A full-stack SAP BTP application with:
- **CAP backend** (Node.js) serving an OData v4 CatalogService
- **SAP UI5 Fiori frontend** (List + Detail views)
- **SAP Approuter** for authentication and routing
- Deployed to **Cloud Foundry** on BTP Trial (org: `cd3f5509trial`, space: `dev`, region: `us10-001`)

## Important Naming (critical ‚Äî must stay consistent)

| Setting | Value |
|---|---|
| `sap.app.id` (manifest.json) | `mybtp.ui` |
| `sap.cloud.service` (manifest.json) | `mybtp` |
| Approuter `welcomeFile` | `/mybtp.ui/index.html` |
| `sap.cloud.service` in mta.yaml destinations | `mybtp` |
| UI5 namespace (index.html, Component.js, controllers, views) | `mybtp.ui` |
| CAP service path | `/catalog/` |

> ‚öÝÔ∏è **Critical routing rule**: The SAP Approuter's dynamic routing extracts ONLY the part BEFORE THE FIRST DOT of the first URL path segment as the "service tag".
> - URL `/mybtp.ui/index.html` ‚Üí service tag = `mybtp` ‚úÖ matches `sap.cloud.service: mybtp`
> - URL `/my.btp.app.ui/index.html` ‚Üí service tag = `my` ‚ùå "Service Tag my is unknown" ‚Üí 503
>
> Therefore: `sap.cloud.service` must equal the text before the first dot in `sap.app.id`.
> With `sap.app.id = "mybtp.ui"` and `sap.cloud.service = "mybtp"` ‚Äî routing works correctly.
>
> The html5-apps-repo-rt serves the app at `/{sap.app.id}/` (full ID as a single URL path segment, dots preserved).

## Deployment Steps

From the project root on **Windows** (requires CF CLI + mbt installed):

```bash
# 1. Build CAP backend
npm run build

# 2. Build UI5 frontend + create zip (already done ‚Äî dist.zip is in resources/)
#    Only redo this if you change frontend source files:
npm run build --prefix app/ui
npm run zip --prefix app/ui
copy app\ui\dist\dist.zip resources\dist.zip

# 3. Build MTA archive
mbt build

# 4. Deploy to CF
cf login   # if not already logged in
cf deploy mta_archives/my-btp-app_1.0.0.mtar
```

App URL after deployment:
`https://cd3f5509trial-dev-my-btp-app-approuter.cfapps.us10-001.hana.ondemand.com`

## Known Issues Fixed

| # | Problem | Root Cause | Fix Applied |
|---|---|---|---|
| 1 | 503 "Service Tag mybtp is unknown" | `sap.cloud.service="mybtp"` but `sap.app.id="my.btp.app.ui"` ‚Äî html5-rt redirected to `/my.btp.app.ui/index.html`, approuter extracted `my` as service tag | First attempt: changed service to `my.btp` and welcomeFile ‚Äî still failed |
| 2 | 503 "Service Tag my is unknown" (after first fix attempt) | URL `/my.btp/app.ui/index.html` ‚Üí approuter still extracts `my` (pre-dot of `my.btp`). Also html5-rt doesn't support multi-segment app IDs ‚Äî it uses `/{sap.app.id}/` as a single path segment | Renamed UI5 namespace + sap.app.id from `my.btp.app.ui` ‚Üí `mybtp.ui`, kept `sap.cloud.service=mybtp`. Now URL is `/mybtp.ui/index.html`, approuter extracts `mybtp` ‚úÖ |
| 3 | 503 "Service Tag mybtp is unknown" (after namespace rename) | Destinations were at `instance` level in the Destination service, but the approuter's dynamic routing queries at `subaccount` level. Also `existing_destinations_policy: ignore` prevented corrections across redeploys. | **Changed `instance` ‚Üí `subaccount` and `ignore` ‚Üí `update` in mta.yaml destination-content** |

> ‚öÝÔ∏è **Critical rule**: In `mta.yaml` destination-content, always use `subaccount:` (not `instance:`). The SAP Approuter's dynamic routing for `html5-apps-repo-rt` routes queries the Destination Service at the **subaccount** level. Instance-level destinations are invisible to it.
> Also use `existing_destinations_policy: update` so destinations are refreshed on every deployment.

**Files changed in namespace rename fix**:
- `app/ui/webapp/manifest.json` ‚Äî sap.app.id, sap.cloud.service, bundleName, viewPath
- `app/ui/webapp/index.html` ‚Äî resourceroots, data-name
- `app/ui/webapp/Component.js` ‚Äî import path, extend namespace
- `app/ui/webapp/controller/List.controller.js` ‚Äî extend namespace
- `app/ui/webapp/controller/Detail.controller.js` ‚Äî extend namespace
- `app/ui/webapp/controller/NotFound.controller.js` ‚Äî extend namespace
- `app/ui/webapp/view/List.view.xml` ‚Äî controllerName
- `app/ui/webapp/view/Detail.view.xml` ‚Äî controllerName
- `app/ui/webapp/view/NotFound.view.xml` ‚Äî controllerName
- `approuter/xs-app.json` ‚Äî welcomeFile ‚Üí `/mybtp.ui/index.html`
- `mta.yaml` ‚Äî destination-content: `instance` ‚Üí `subaccount`, `ignore` ‚Üí `update`
- `resources/dist.zip` ‚Äî rebuilt and copied after all source changes
